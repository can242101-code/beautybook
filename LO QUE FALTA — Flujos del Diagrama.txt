Flujo 2: Consultorio configura sus horarios y tratamientos

Consultorio   Frontend         Laravel 13       Redis           PostgreSQL
    |              |                |               |                |
    |--inicia sesión en su panel--->|               |                |
    |              |--POST /login------------------>|                |
    |              |                |--valida credenciales---------->|
    |              |                |<--datos del consultorio--------|
    |              |<--token de sesión activo--------|               |
    |              |                |               |                |
    |--registra tratamiento-------->|               |                |
    |  (nombre, duración, precio)   |               |                |
    |              |--POST /tratamientos----------->|                |
    |              |                |--INSERT tratamiento----------->|
    |              |                |<--tratamiento guardado---------|
    |              |<--confirmación: tratamiento creado--------------|
    |              |                |               |                |
    |--configura horarios de------->|               |                |
    |  atención (días y horas)      |               |                |
    |              |--POST /horarios--------------->|                |
    |              |                |--INSERT horarios-------------->|
    |              |                |<--horarios guardados-----------|
    |              |                |--actualiza disponibilidad----->|
    |              |                |<--caché actualizado------------|
    |              |<--confirmación: horarios configurados-----------|
    |<--panel muestra agenda actualizada con los nuevos bloques------|

┌──────┬──────────────────────────────────────────┬───────────────────┐

│ 2    │ Frontend envía credenciales al backend   │ Petición HTTP     │
│      │                                          │ POST              │
├──────┼──────────────────────────────────────────┼───────────────────┤
│ 3    │ Backend valida y retorna token de sesión │ Respuesta HTTP    │
├──────┼──────────────────────────────────────────┼───────────────────┤
│ 4    │ Consultorio registra un tratamiento con  │ Acción del actor  │
│      │ nombre, duración y precio                │                   │
├──────┼──────────────────────────────────────────┼───────────────────┤
│ 5    │ Frontend envía los datos del tratamiento │ Petición HTTP     │
│      │  al backend                              │ POST              │
├──────┼──────────────────────────────────────────┼───────────────────┤
│ 6    │ Backend inserta el tratamiento en        │ Escritura en base │
│      │ PostgreSQL                               │  de datos         │
├──────┼──────────────────────────────────────────┼───────────────────┤
│ 7    │ Consultorio configura sus días y horas   │ Acción del actor  │
│      │ de atención                              │                   │
├──────┼──────────────────────────────────────────┼───────────────────┤
│ 8    │ Frontend envía la configuración de       │ Petición HTTP     │
│      │ horarios al backend                      │ POST              │
├──────┼──────────────────────────────────────────┼───────────────────┤
│ 9    │ Backend inserta los horarios en          │ Escritura en base │
│      │ PostgreSQL                               │  de datos         │
├──────┼──────────────────────────────────────────┼───────────────────┤
│ 10   │ Backend actualiza la disponibilidad en   │ Escritura en      │
│      │ Redis con los nuevos bloques             │ memoria           │
├──────┼──────────────────────────────────────────┼───────────────────┤
│ 11   │ Frontend muestra la agenda del           │ Respuesta al      │
│      │ consultorio con los cambios aplicados    │ actor             │
└──────┴──────────────────────────────────────────┴───────────────────┘

Relación con el flujo del Paciente: Los horarios y tratamientos que el Consultorio registra en este flujo son exactamente los que el Paciente consulta en el paso 2 del Flujo 1 (GET /disponibilidad). Si el Consultorio no configura sus horarios, el Paciente no verá ningún espacio disponible. Redis actúa como el puente entre ambos flujos.

Flujo 3: Gestor del sistema administra un consultorio registrado

Gestor         Frontend         Laravel 13                  PostgreSQL
   |               |                |                             |
   |--inicia sesión como gestor---->|                             |
   |               |--POST /login (rol: gestor)----------------->  |
   |               |                |--valida rol de gestor------>|
   |               |                |<--acceso confirmado---------|
   |               |<--panel de administración disponible---------|
   |               |                |                             |
   |--consulta lista de------------>|                             |
   |  consultorios registrados      |                             |
   |               |--GET /admin/consultorios------------------->|
   |               |                |--SELECT consultorios------->|
   |               |                |<--lista con estado y plan---|
   |               |<--muestra tabla de consultorios y estados----|
   |               |                |                             |
   |--selecciona consultorio------->|                             |
   |  con membrecía vencida         |                             |
   |               |--GET /admin/consultorios/{id}-------------->|
   |               |                |--SELECT detalle consultorio>|
   |               |                |<--datos y fecha de vencim---|
   |               |<--muestra detalle con alerta de vencimiento--|
   |               |                |                             |
   |--actualiza plan o vence------->|                             |
   |  membrecía manualmente         |                             |
   |               |--PUT /admin/membrecia/{id}----------------->|
   |               |                |--UPDATE membrecía---------->|
   |               |                |<--registro actualizado------|
   |               |<--confirmación: membrecía actualizada--------|
   |<--panel refleja el nuevo estado del consultorio--------------|

Relación con los otros flujos: Cuando el Gestor vence o bloquea una membrecía en este flujo, el caso de verificación CV05 del sistema se activa automáticamente: el Consultorio queda inhabilitado para recibir nuevas citas hasta que renueve su plan. Esto protege el modelo de negocio de BeautyBook sin requerir intervención manual en cada agendamiento.
